# Vangate 网站设计系统规范

## 品牌背景

**产品**: VanGATE - 汽车配件与设备B2B/B2C平台
**市场**: 北美地区（加拿大为主）
**用户**: 车主、汽车维修店、商业车队
**核心价值**: 专业、可靠、全面的汽车配件供应

---

## 三种设计风格

### 风格一：现代极简风 (Modern Minimal)

**定位**: 高端专业、清新舒适、信任感强
**适用人群**: 高端车主、维修连锁企业
**美学方向**: 克制极简化、大量留白、精致细节

```
色彩系统:
- Primary: #1E3A5F (深海蓝 - 信任)
- Secondary: #F8FAFC (云白 - 干净)
- Accent: #0EA5E9 (天空蓝 - 活力)
- Text: #1E293B (墨灰)
- Muted: #64748B (雾灰)
- Success: #10B981
- Background: #FFFFFF

字体系统:
- 标题: Plus Jakarta Sans (600, 700)
- 正文: Inter (400, 500)
- 字重对比: 标题 Bold, 正文 Regular

间距系统: 8px基准
- xs: 4px, sm: 8px, md: 16px, lg: 24px, xl: 32px, 2xl: 48px, 3xl: 64px
```

### 风格二：工业专业风 (Industrial Professional)

**定位**: 专业可靠、稳重高效、功能导向
**适用人群**: 商用客户、维修技师、车队管理
**美学方向**: 工业实用主义、深色主题、信息密度高

```
色彩系统:
- Primary: #F97316 (工业橙 - 活力/警示)
- Secondary: #18181B (深黑 - 专业)
- Surface: #27272A (碳灰 - 层次)
- Text: #FAFAFA (纯白)
- Muted: #A1A1AA (灰白)
- Border: #3F3F46
- Accent: #FBBF24 (琥珀黄)

字体系统:
- 标题: Bebas Neue / Oswald (700) - 工业感
- 正文: Source Sans 3 (400, 600)

间距系统: 8px基准
```

### 风格三：活力现代风 (Vibrant Contemporary)

**定位**: 年轻活力、现代科技、品牌突出
**适用人群**: 年轻车主、汽车爱好者、改装市场
**美学方向**: 色彩大胆、动效丰富、互动感强

```
色彩系统:
- Primary: #2563EB (电光蓝 - 科技)
- Secondary: #DC2626 (赛道红 - 激情)
- Accent: #10B981 (速度绿 - 环保)
- Neutral: #F1F5F9
- Text: #0F172A
- Gradient: linear-gradient(135deg, #2563EB, #7C3AED)

字体系统:
- 标题: Outfit (600, 700) - 现代几何
- 正文: DM Sans (400, 500) - 圆润易读

间距系统: 4px基准 (更紧凑)
```

## 组件规范

### 导航栏
- **极简风**: 白色背景、细线分隔、品牌色logo
- **工业风**: 深色背景、橙色强调、粗体文字
- **活力风**: 渐变背景、白色文字、圆角按钮

### 产品卡片
- 极简风: 纯白背景、细边框、悬浮阴影
- 工业风: 深色背景、橙色标签、粗体价格
- 活力风: 渐变边框、悬停动画、快捷操作

### 按钮
- 极简风: 描边按钮、圆角sm、过渡柔和
- 工业风: 实心按钮、直角或微圆角、厚重感
- 活力风: 渐变按钮、大圆角、阴影强调

### CTA (行动召唤)
- 极简风: "联系我们" - 细长按钮、蓝色
- 工业风: "获取报价" - 橙色大按钮、hover加亮
- 活力风: "立即选购" - 渐变背景、动画效果

## 移动端适配规范 (Mobile-First Responsive Design)

### 设计决策

**采用方案**: 单版本自适应（响应式设计）
- Google 推荐的最佳实践
- 单一 URL，SEO 友好
- 维护成本低，用户体验一致

### 响应式断点详细规范

```
┌─────────────────────────────────────────────────────────────┐
│  Mobile (< 640px)    │  Tablet (640-1023px)                 │
│  单列布局             │  双列布局                            │
│  汉堡菜单             │  简化导航                            │
├─────────────────────────────────────────────────────────────┤
│  Tablet (640-1023px) │  Desktop (1024-1279px)                │
│  双列布局             │  三列布局                            │
│  简化导航             │  完整导航                            │
├─────────────────────────────────────────────────────────────┤
│  Desktop (1280px+)    │  Large (1536px+)                      │
│  四列布局             │  最大内容宽度 1400px                  │
└─────────────────────────────────────────────────────────────┘
```

### 移动端组件规范

#### 1. 导航栏 (Navbar)
```

```

Tablet+:

- 导航按钮: 48x48px

```
Mobile (< 640px):  1 列
Tablet (640-1023px): 2 列

```

#### 4. 表单 (Forms)
```
Mobile:
- 输入框高度: 48px
- 按钮高度: 48px (主按钮), 44px (次要按钮)
- 内边距: 16px
- 标签字体: 14px

Tablet+:
- 输入框高度: 56px
- 按钮高度: 56px (主按钮), 48px (次要按钮)
- 内边距: 20px
- 标签字体: 16px
```

#### 5. 图片处理
```
背景图:
- Mobile: 高度 50vh, cover 裁切
- Tablet+: 高度 70vh, cover 裁切

产品图片:
- Mobile: 宽度 100%, 4:3 比例
- 使用 srcset 适配不同分辨率
```

### 触摸友好规范

1. **点击区域**: 所有可交互元素最小 44x44px
2. **间距**: 相邻可点击元素间距至少 8px
3. **禁用 hover-only**: 功能不能仅依赖 hover 触发
4. **滚动**: 水平滚动区域标记 scroll-snap

### 性能优化

1. **首屏优先**: 关键 CSS 内联
2. **图片懒加载**: 使用 loading="lazy"
3. **字体显示**: font-display: swap
4. **动画**: 使用 transform/opacity，避免重排

## 手机适配要点

1. **触摸优化**: 所有可点击元素最小44x44px
2. **手势友好**: 支持滑动手势、拖拽操作
3. **简化导航**: 汉堡菜单、底部导航栏
4. **单手操作**: 常用操作在拇指区
5. **快速加载**: 首屏优先、懒加载图片
